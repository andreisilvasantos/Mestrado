def binario16_para_decimal(vetor, i_expoente=2, i_mantissa=6):
    sinal = -1 if vetor[0] == 1 else 1 # Guadra o sinal do nmero
    sinal_expoente = -1 if vetor[1] == 1 else 1 # Guarda o sinal do expoente

    expoente = 0
    for i in range(i_expoente, i_mantissa):
        expoente += vetor[i] * 2 ** (i_mantissa - 1 - i)  # Calculo iterativo do expoente de n bits
    expoente *= sinal_expoente # Expoente recebe o sinal aps converso

    mantissa = 0
    for i in range(i_mantissa, len(vetor)):
        mantissa += vetor[i] * 2 ** (i_mantissa - 1 - i) # Calculo iterativo da mantissa de n bits

    return sinal * mantissa * (2 ** expoente)

vetor = [0,1,0,0,1,1,1,0,1,0,1,1,0,0,0,1] ### Vetor dado
print("Resposta: Nmero decimal representado :", binario16_para_decimal(vetor))

e) Utilize os resultados anteriores para explicar porque apenas um subconjunto dos nmeros reais pode ser representado pelo computador.

Limitao da mantissa: Com 10 bits disponveis para a mantissa, podemos representar at 2 = 1024 diferentes combinaes binrias. Isso nos permite representar a parte fracionria dos nmeros com at 10 bits de preciso. No entanto, essa preciso limitada e nmeros que exigem mais de 10 bits para representar sua parte fracionria precisaro ser arredondados ou truncados, resultando em uma perda de preciso.

Limitao do expoente com sinal: Com 4 bits disponveis para o expoente (incluindo 1 bit para o sinal do expoente), podemos representar nmeros muito grandes ou muito pequenos. O bit de sinal do expoente nos permite representar nmeros positivos e negativos. Com 4 bits, podemos representar nmeros de aproximadamente -8 a 7.

Representao finita: Como todos os nmeros de ponto flutuante so representados em uma quantidade finita de bits, h um limite para a preciso e a faixa de valores que podemos representar. Com um total de 16 bits (10 para a mantissa, 1 para o sinal, 1 para o sinal do expoente e 4 para o expoente), estamos limitados a representar nmeros de ponto flutuante em uma faixa especfica e com uma preciso limitada.

Assim, mesmo com um formato de ponto flutuante de 16 bits, apenas um subconjunto finito e discreto dos nmeros reais pode ser representado com preciso, devido s limitaes no tamanho da mantissa e do expoente, bem como representao finita dos nmeros em um computador.

    euler_estimado = 1

    if x == 0: ### Caso Base

    for i in range(1, x + 1): ### Caso Geral
        euler_estimado += 1 / math.factorial(i)
    vetor_resultados_euler.append(euler_estimado)
